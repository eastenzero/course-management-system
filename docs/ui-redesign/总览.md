## UI 重设计总览（教师端/学生端）

本总览汇聚了视觉基调、主题方案、组件规范与页面改造清单的关键要点，提供落地流程、验收指标与链接占位，便于设计、前端、产品协同推进。

---

## 文档与素材结构

- 设计规范.md：视觉基调、玻璃拟态参数、动效、无障碍、性能与降级
- 主题方案.md：莫奈/莫兰迪预设主题、功能色、明暗模式与对比度策略
- 组件规范.md：组件外观与四态、玻璃化适配、动效与可用性
- 页面改造清单.md：教师/学生端页面级改造要点、优先级、排期与验收
- design-tokens.example.json：Design Tokens 示例结构（可直接接入）

---

## 快速落地流程（建议）

1) 确认主题与色板
- 在 主题方案.md 中选定 1–2 套主推主题（教师端偏 Morandi / 学生端偏 Monet）
- 明暗模式与功能色按对比度策略校验（AA）

2) 建立 Design Tokens
- 以 design-tokens.example.json 为模板，创建项目实际 tokens（命名与层级保持一致）
- 规划色彩、中性层、玻璃 alpha、圆角、阴影、模糊与动效变量

3) 注入样式层
- 将 Tokens 映射为 CSS 变量（:root 与 [data-theme]）
- 实现主题切换（设置入口 + 首次引导 3 步），持久化与首屏防闪烁

4) 组件外观改造
- 依据 组件规范.md，为按钮/输入/卡片/表格/导航等完成玻璃化与四态
- 校验焦点可见、对比度、键盘可达与“减少动效”模式

5) 页面级替换
- 参考 页面改造清单.md：按 P0/P1/P2 优先级逐步替换
- 学生端强化亲和插画与奖励动效，教师端强化数据可读性

6) 性能与降级
- 控制模糊层与面积；不支持 backdrop-filter 时降级为半透明 + 细描边 + 阴影
- 图像与插画使用 WebP/AVIF，懒加载；避免过强发光/阴影

7) 验收与灰度
- 执行无障碍与性能基线验证；对关键任务做可用性走查
- 小流量灰度上线，收集 NPS/CSAT 与性能指标后修订规范 v2

## 主题与对比度策略（摘要）

- Monet：水彩柔光、轻高光；Morandi：低饱和灰调、静谧克制
- 文本对比度：正文 ≥ 4.5:1，标题 ≥ 3:1；动态主题需自动微调亮度/饱和度
- 图片/模糊背景上的文字需叠加渐变遮罩或半透明底

---

## Glassmorphism 参数（摘要）

- 模糊：8/16/24px（sm/md/lg）
- 玻璃 Alpha：Light 0.55–0.68；Dark 0.28–0.40
- 高光描边：α≈0.30–0.38，1px 内外发光结合；阴影：y=8–24px，blur=24–48px，α≈0.08–0.12
- 圆角：按钮 10–12、卡片 16、弹窗/抽屉 20–24；8pt 栅格留白

---

## 浏览器兼容与降级（摘要）

- 支持 backdrop-filter：启用真实模糊；否则半透明 + 分层阴影 + 细描边
- 降低模糊层数量与面积；复用组件层，避免滚动抖动

---

## 验收与度量（摘要）

- 可用性：关键任务完成率 ≥ 95%，关键路径时长下降 10–20%
- 视觉一致性：组件均通过规范审查；对比度 AA 达标
- 性能：LCP/INP 无显著下降；低配设备降级稳定
- 无障碍：键盘导航覆盖 100%，焦点可见；“减少动效”有效
