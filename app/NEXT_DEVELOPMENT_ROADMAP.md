## 🎨 提示词1: 前端用户体验现代化 (P0)

1. **界面现代化**: 提升视觉设计，优化交互体验
2. **性能优化**: 减少加载时间，提升响应速度  
3. **用户友好**: 简化操作流程，提高易用性
4. **移动适配**: 确保在各种设备上的良好体验

- **UI组件**: 使用现有毛玻璃效果系统，确保视觉一致性
- **状态管理**: 基于Redux Toolkit进行状态优化
- **性能**: 实现虚拟滚动、懒加载、代码分割
- **响应式**: 优化移动端和平板适配

### 具体实施要求
1. **首页重构**: 
   - 设计现代化仪表板，展示关键数据指标

4. **性能集成测试**:
   - 测试大量数据下的系统响应性能
   - 验证并发用户访问时的系统稳定性
   - 检查内存使用和CPU负载的合理性

### 质量标准
- **API成功率**: 接口调用成功率 > 99.9%
- **响应时间**: API响应时间 < 500ms，排课算法 < 30秒

## ⚡ 提示词3: 性能优化与扩展性 (P1)

### 任务背景
系统已具备完整功能，现需要针对性能进行深度优化，确保能够支持大规模用户使用。特别是在处理大量课程数据、复杂排课算法计算、并发用户访问等方面需要优化。

### 核心目标
1. **算法性能**: 优化排课算法执行效率，支持更大规模数据
2. **前端性能**: 优化页面加载速度和交互响应性
3. **后端性能**: 优化数据库查询和API响应速度
4. **系统扩展性**: 设计支持水平扩展的架构

### 技术架构要求
- **缓存策略**: 实现多层缓存(Redis、浏览器缓存、CDN)
- **数据库优化**: 索引优化、查询优化、连接池管理
- **算法优化**: 并行计算、内存优化、算法调优
- **前端优化**: 代码分割、懒加载、虚拟化渲染

### 具体实施要求
1. **算法性能优化**:
   - 实现遗传算法的并行化处理
   - 优化约束检查的计算效率
   - 添加算法执行进度监控和中断机制
   - 实现算法结果缓存，避免重复计算

2. **数据库性能优化**:
   - 分析慢查询，添加必要的数据库索引
   - 实现查询结果缓存和数据预取
   - 优化数据模型，减少不必要的关联查询
   - 实现数据库连接池和读写分离

3. **前端性能优化**:
   - 实现组件级代码分割和按需加载
   - 优化大列表渲染，使用虚拟滚动
   - 实现图片懒加载和渐进式加载
   - 添加Service Worker实现离线缓存

4. **API性能优化**:
   - 实现API响应压缩和缓存
   - 优化序列化和反序列化性能
   - 实现API限流和熔断机制
   - 添加API性能监控和告警

### 质量标准
- **算法性能**: 处理1000+课程的排课时间 < 60秒
- **页面加载**: 首屏加载时间 < 3秒，后续页面 < 1秒
- **API响应**: 90%的API请求响应时间 < 200ms
- **并发处理**: 支持500+并发用户同时访问
- **内存使用**: 服务器内存使用率 < 80%

### 验证方式
- 使用压力测试工具进行负载测试
- 监控系统资源使用情况和性能指标
- 进行真实场景下的性能测试
- 使用性能分析工具识别瓶颈

## 🤖 提示词4: 智能化功能增强 (P1)

### 任务背景
当前排课算法已经十分成熟，智能数据生成器工作良好。在此基础上，可以进一步增强系统的智能化水平，引入机器学习、预测分析等先进技术，提升用户体验和系统效能。

### 核心目标
1. **预测分析**: 基于历史数据预测课程需求和资源使用
2. **智能推荐**: 为用户提供个性化的课程和时间推荐
3. **自动优化**: 系统自学习，持续优化排课质量
4. **异常检测**: 智能识别异常数据和潜在问题

### 技术架构要求
- **机器学习**: 集成scikit-learn、TensorFlow等ML框架
- **数据分析**: 使用pandas、numpy进行数据分析
- **实时计算**: 实现流式数据处理和实时预测
- **智能缓存**: 基于用户行为的智能缓存策略

### 具体实施要求
1. **智能课程推荐系统**:
   - 基于学生历史选课数据推荐相关课程
   - 根据教师专业和历史授课情况推荐适合的课程
   - 实现协同过滤和内容推荐的混合推荐算法
   - 提供推荐理由和可信度评分

2. **预测分析模块**:
   - 预测各课程的选课人数，辅助资源规划
   - 分析教师和教室的使用规律，优化资源配置
   - 预测系统负载，提前进行扩容准备
   - 基于季节和历史趋势预测课程需求

3. **智能排课优化**:
   - 分析历史排课结果，总结最佳实践
   - 自动调整算法参数，提升排课质量
   - 学习用户反馈，持续优化排课策略
   - 实现A/B测试框架，验证优化效果

4. **异常检测与预警**:
   - 检测异常的选课行为和数据录入
   - 监控系统性能异常和潜在故障
   - 预警课程冲突和资源短缺
   - 智能识别数据质量问题

### 质量标准
- **推荐准确率**: 课程推荐接受率 > 60%
- **预测精度**: 选课人数预测误差 < 15%
- **优化效果**: 排课质量评分提升 > 10%
- **异常检测**: 异常检测准确率 > 85%
- **响应时间**: 智能分析响应时间 < 5秒

### 验证方式
- 使用历史数据进行回测验证
- 进行A/B测试比较优化效果
- 收集用户反馈评估推荐质量
- 监控系统指标验证改进效果
