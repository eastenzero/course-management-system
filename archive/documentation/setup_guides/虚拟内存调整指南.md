# 虚拟内存调整到G盘操作指南

## 方法一：使用PowerShell脚本（推荐）

1. **以管理员身份运行PowerShell**
   - 按 `Win + X`，选择"Windows PowerShell (管理员)"
   - 或右键点击开始按钮，选择"终端(管理员)"

2. **执行调整脚本**
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   .\adjust_pagefile.ps1
   ```

3. **重启计算机**
   - 设置完成后必须重启才能生效

## 方法二：通过系统设置手动调整

1. **打开系统属性**
   - 按 `Win + R`，输入 `sysdm.cpl`，按回车
   - 或者：控制面板 → 系统 → 高级系统设置

2. **进入性能设置**
   - 点击"高级"选项卡
   - 在"性能"部分点击"设置"按钮

3. **调整虚拟内存**
   - 点击"高级"选项卡
   - 在"虚拟内存"部分点击"更改"按钮

4. **配置新的虚拟内存**
   - 取消勾选"自动管理所有驱动器的分页文件大小"
   - 选择D盘，选择"无分页文件"，点击"设置"
   - 选择G盘，选择"自定义大小"
   - 初始大小：16384 MB (16GB)
   - 最大大小：32768 MB (32GB)
   - 点击"设置"，然后"确定"

5. **重启计算机**

## 预期效果

调整完成后：
- 虚拟内存将使用G盘而不是D盘
- Git操作时的硬盘读写活动将转移到G盘
- D盘的异常读写问题应该得到解决
- 系统性能可能会有所改善

## 验证设置

重启后可以通过以下命令验证：
```powershell
wmic pagefile list /format:list
```

应该看到虚拟内存文件位于 `G:\pagefile.sys`

## 注意事项

1. **备份重要数据**：虽然这个操作相对安全，但建议先备份重要数据
2. **监控G盘空间**：确保G盘有足够空间容纳虚拟内存文件
3. **性能监控**：调整后可以监控系统性能，确保没有负面影响
4. **回滚方案**：如果有问题，可以按相同步骤将虚拟内存调整回其他位置

## 推荐的虚拟内存大小

基于您的32GB物理内存：
- **保守设置**：初始16GB，最大32GB
- **性能优化**：初始8GB，最大16GB（如果很少遇到内存不足）
- **最大兼容**：初始32GB，最大48GB（如果经常处理大型数据）
